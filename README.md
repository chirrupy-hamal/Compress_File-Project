文件压缩是指按照一定的算法对文件中的数据重新进行组织，在不丢失有用信息的前提下，缩减数据量以减少存储空间。
提高其存储、传输和处理（例如，删除）效率。

为什么需要压缩？
1、节省存储空间。
2、节省流量。缩短传输时间，进而提高网络通信效率。
3、起到一定程度的加密作用，增强数据在网络传输过程中的安全性。将数据用另外的一种编码格式进行改写。

压缩的分类
1、无损压缩：压缩文件经解压缩完全还原成原文件。
2、有损压缩：压缩文件经解压缩后与原文件间存在差异。图片/视频

压缩的本质
压缩的目的是让文件变小，减少文件的存储空间。压缩率 = 压缩文件大小/原文件大小
那怎么才能让文件变小呢？
1、陕西科技大学->陕科大。找到一个更短的、大家都知道的进行替换。但是可能性不是很高，因为文件是各种各样的。
2、ABBBCCCCCDDDDDDD这样一个文件以字节流的形式被储到磁盘中，16字节。
   ->能否？找到一个更短的编码(小于8比特)替换原编码。
     4种字符，2个比特位就可以了，00代表'A'  01代表'B'  10代表'C'  11代表'D'
   ->00010101 10101010 10111111 11111111 4字节
   ->解压缩ABBB CCCC CDDD DDDD
   上述的编码是等长编码，简单的等长编码很容易被别人破解文件的规则。
   不等长编码(基于Huffman树的哈夫曼编码)，100代表'A'  101代表'B'  11代表'C'  0代表'D'
   ->10010110 11011111 11111100 000000___
3、(距离,长度)对，先判断长度，再判断距离。LZ77
   mnoabczxyuvwabc123456abczxydefgh->mnoabczxyuvw(9,3)123456(18,6)defgh，在正式的压缩中，括号和逗号是没有的。

基于Huffman树(https://blog.csdn.net/chirrupy_hamal/article/details/93981514)的哈夫曼编码：
1、获取原文件中每个字符出现的次数(权值)；A:1，B:3，C:5，D:7。
2、构造Hufman树；
   (1)小堆，管理二叉树森林。
   (2)优先级队列priority_queue就是堆。但默认是大堆，用仿函数解决。
3、获取Huffman编码(左0右1);
4、用Huffman编码重新对原文件进行改写。

解压缩
1、提供一份压缩文件 .rar；
2、获取解压缩的信息；
3、还原Huffman树；
4、解压缩。
